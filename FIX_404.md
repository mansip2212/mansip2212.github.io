# Fix 404 Error - Quick Guide

## The Problem
You're getting a 404 because you're accessing the wrong URL or the deployment isn't complete.

## Solution 1: Use the Correct URL

Since your repository is `portfolio-v2`, your site is at:
**https://mansip2212.github.io/portfolio-v2/**

❌ **NOT** at: `https://mansip2212.github.io/home` or `https://mansip2212.github.io/`

## Solution 2: Deploy to Root (if you want it at mansip2212.github.io)

If you want your site at the root URL (`https://mansip2212.github.io/`):

### Option A: Rename Repository
1. Go to: https://github.com/mansip2212/portfolio-v2/settings
2. Scroll to "Repository name"
3. Rename from `portfolio-v2` to `mansip2212.github.io`
4. Update `vite.config.js` - change `base: '/portfolio-v2/'` to `base: '/'`
5. Push the change

### Option B: Keep Repository Name, Change Base Path
1. Update `vite.config.js` - change `base: '/portfolio-v2/'` to `base: '/'`
2. Update remote URL if needed
3. Push changes

## Check Deployment Status

1. Go to: https://github.com/mansip2212/portfolio-v2/actions
2. Check if the latest workflow run completed successfully (green checkmark ✅)
3. If it failed, check the error logs

## Verify GitHub Pages is Enabled

1. Go to: https://github.com/mansip2212/portfolio-v2/settings/pages
2. Make sure "Source" is set to **GitHub Actions**
3. Save if needed
